Create three new files and update JBalance.java:

**src/main/java/com/pweg0/jbalance/data/db/BalanceRepository.java:**

Must contain these methods (all use raw JDBC via DataSource):
- `long getBalance(UUID uuid)` — SELECT balance WHERE uuid=?; returns -1L if not found
- `boolean initPlayerIfAbsent(UUID uuid, String displayName, long startingBalance)` — dialect-aware INSERT: MySQL uses `INSERT IGNORE INTO`, SQLite uses `INSERT OR IGNORE INTO`. The `isMysql` boolean is passed via constructor. Returns true if row was inserted (new player).
- `boolean adjustBalance(UUID uuid, long delta)` — For positive delta: `UPDATE jbalance_players SET balance = balance + ? WHERE uuid = ?`. For negative delta: `UPDATE jbalance_players SET balance = balance + ? WHERE uuid = ? AND balance >= ?` (sets param 3 to `-delta`). Returns `executeUpdate() == 1`.
- `boolean transfer(UUID from, UUID to, long amount)` — Uses a single Connection with `setAutoCommit(false)`. Deduct: `UPDATE ... SET balance = balance - ? WHERE uuid = ? AND balance >= ?`. If rows affected != 1, rollback and return false. Credit: `UPDATE ... SET balance = balance + ? WHERE uuid = ?`. Commit. In catch block: rollback. In finally: `setAutoCommit(true)`.

Constructor takes `DataSource dataSource` and `boolean isMysql`.

**src/main/java/com/pweg0/jbalance/service/EconomyService.java:**

Must contain:
- A static `ExecutorService DB_EXECUTOR = Executors.newFixedThreadPool(2, r -> { Thread t = new Thread(r, "JBalance-DB"); t.setDaemon(true); return t; })` — daemon threads that do not prevent JVM shutdown
- A `ConcurrentHashMap<UUID, AtomicBoolean> inFlight` for per-player transfer locking
- A `BalanceRepository repo` field
- A static `EconomyService instance` field with `getInstance()` accessor
- `CompletableFuture<Long> getBalance(UUID playerId)` — supplyAsync on DB_EXECUTOR
- `CompletableFuture<Boolean> give(UUID playerId, long amount)` — calls `repo.adjustBalance(uuid, +amount)` async
- `CompletableFuture<Boolean> take(UUID playerId, long amount)` — calls `repo.adjustBalance(uuid, -amount)` async
- `CompletableFuture<Boolean> transfer(UUID from, UUID to, long amount)` — acquires per-player in-flight lock via `inFlight.computeIfAbsent(from, k -> new AtomicBoolean(false))` + `compareAndSet(false, true)`; if lock fails, return `CompletableFuture.completedFuture(false)`. Calls `repo.transfer()` in supplyAsync; releases lock in finally block.
- `CompletableFuture<Boolean> initPlayerIfAbsent(UUID uuid, String displayName)` — reads `JBalanceConfig.STARTING_BALANCE.get()` on the calling thread (game thread), then passes the value into async `repo.initPlayerIfAbsent(uuid, displayName, startingBalance)`
- `void shutdown()` — shuts down DB_EXECUTOR with `shutdownNow()`

CRITICAL: `initPlayerIfAbsent` reads config on the GAME THREAD before going async. Do NOT call `ConfigValue.get()` from the DB_EXECUTOR thread.

private static void onServerAboutToStart(ServerAboutToStartEvent event) {
    dbManager = new DatabaseManager();
    economyService = new EconomyService(dbManager);
    LOGGER.info("[JBalance] Database and economy service initialized");
}

private static void onServerStopping(ServerStoppingEvent event) {
    if (economyService != null) economyService.shutdown();
    if (dbManager != null) dbManager.shutdown();
    LOGGER.info("[JBalance] Database connections closed");
}
