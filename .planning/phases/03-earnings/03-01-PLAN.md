Also add the static helper method to parse the mob rewards list into a Map:

```java
/**
 * Parses the MOB_KILL_REWARDS config list into a Map of entity type key -> reward amount.
 * MUST be called on the game thread (config values are not thread-safe).
 */
public static java.util.Map<String, Long> parsedMobRewards() {
    java.util.Map<String, Long> map = new java.util.HashMap<>();
    for (Object entry : MOB_KILL_REWARDS.get()) {
        String[] parts = entry.toString().split("=", 2);
        if (parts.length == 2) {
            try {
                map.put(parts[0].trim(), Long.parseLong(parts[1].trim()));
            } catch (NumberFormatException ignored) {}
        }
    }
    return map;
}
```

IMPORTANT: Use `defineListAllowEmpty` (NOT `defineList`) to avoid NeoForge config infinite correction loop (Issue #1768). Use `defineInRange` for the notification interval (same pattern as all other numeric config values in this project).

`accumulateKill(UUID uuid, long reward)`:

**4. `onServerTick(ServerTickEvent.Post event)`**
- Increment `flushTickCounter`
- Read notification interval on game thread: `long intervalSeconds = JBalanceConfig.KILL_NOTIFICATION_INTERVAL.get();`
- If `intervalSeconds <= 0`, flush every tick (immediate mode)
- Otherwise, flush when `flushTickCounter >= intervalSeconds * 20` (20 ticks/second), then reset counter
- Flush logic: iterate `pendingCoins`, for each UUID:
  - Get `ServerPlayer` via `event.getServer().getPlayerList().getPlayer(uuid)` (returns null if offline)
  - If player online: send batched notification message, call `EconomyService.getInstance().give(uuid, coins)`
  - Clear entries from both maps after processing
- Notification message format (PT-BR, locked decision):
  `"§6[JBalance] §7Voce recebeu §6" + CurrencyFormatter.formatBalance(totalCoins) + " §7por matar §6" + killCount + " §7mobs"`
- Import: `net.neoforged.neoforge.event.tick.ServerTickEvent`
